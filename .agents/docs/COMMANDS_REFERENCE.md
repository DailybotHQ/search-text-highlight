## Repo-specific slash commands

These commands are tuned for `search-text-highlight`. They live in [`.agents/skills/`](../skills/) and [`.agents/commands/`](../commands/).

| Command            | Type    | One-liner                                                                               |
| ------------------ | ------- | --------------------------------------------------------------------------------------- |
| `/add-option`      | skill   | Add a new option to `OptionsType` end-to-end (types, validators, defaults, tests, docs) |
| `/add-feature`     | skill   | Add a new method to `searchTextHL` — major version bump                                 |
| `/write-tests`     | skill   | Author Vitest tests for the current change                                              |
| `/fix-build`       | skill   | Diagnose and repair a failing Vite / tsc build                                          |
| `/lint-fix`        | skill   | Run Biome with `--write`                                                                |
| `/bump-deps`       | skill   | Update `package.json` safely; respects `.ncurc.json`                                    |
| `/release`         | skill   | Run the release workflow and verify prerequisites                                       |
| `/devcontainer-up` | skill   | Spin up the local Docker dev environment                                                |
| `/fork-rebrand`    | skill   | Walk a fresh fork through name, npm scope, repo URL, license                            |
| `/verify`          | command | Run the full pre-push check chain                                                       |
| `/pack-check`      | command | Inspect what `pnpm pack` would publish                                                  |
| `/ci-reproduce`    | command | Reproduce a failing CI workflow step locally                                            |
| `/reset-env`       | command | Clear `node_modules` + caches and reinstall                                             |

## Deep Work Plan delegators

Thin delegators that forward to the installed [DWP skill pack](https://github.com/DailybotHQ/deepworkplan-skill) under `~/.claude/skills/deepworkplan-*`.

| Command                | Delegates to           | Purpose                                                                                    |
| ---------------------- | ---------------------- | ------------------------------------------------------------------------------------------ |
| `/dwp-create <goal>`   | `deepworkplan-create`  | Decompose a goal into a numbered plan with validation gates; output lands in `.dwp/plans/` |
| `/dwp-execute`         | `deepworkplan-execute` | Run a plan task-by-task, validate each gate, update progress                               |
| `/dwp-refine`          | `deepworkplan-refine`  | Add, remove, or reorder tasks while preserving completed work                              |
| `/dwp-resume`          | `deepworkplan-resume`  | Reconstruct state and continue an interrupted plan                                         |
| `/dwp-status`          | `deepworkplan-status`  | Report progress on the active plan without making changes                                  |
| `/dwp-verify`          | `deepworkplan-verify`  | Objective pass/fail conformance report (read-only)                                         |
| `/dwp-onboard`         | `deepworkplan-onboard` | Re-run the AI-first onboarding non-destructively                                           |
| `/skill-create <name>` | `deepworkplan-author`  | Author a new skill for this repo                                                           |
| `/agent-create <name>` | `deepworkplan-author`  | Author a new agent for this repo                                                           |

## How to invoke

| Host                                     | Prefix              | Example                            |
| ---------------------------------------- | ------------------- | ---------------------------------- |
| Claude Code                              | `/`                 | `/dwp-create add wholeWord option` |
| OpenAI Codex / Cursor / Gemini / Copilot | `#` (or plain name) | `#dwp-create add wholeWord option` |

For DWP commands, the delegator file points at the installed sub-skill — the agent reads the installed skill's `SKILL.md` and follows it. If the skill is not installed, the delegator stops and tells the user how to install it (see [`AGENTS.md`](../../AGENTS.md) "Skills & Agents").

## Output conventions

- **Plans** generated by `/dwp-create` and `/dwp-refine` are written to `.dwp/plans/<slug>.md`.
- **Drafts** (intermediate / unconfirmed plans) are written to `.dwp/drafts/`.
- Both directories are gitignored except for `.gitkeep` markers — plans are runtime artifacts, not source code.
